## 1. 프로젝트 표준 디렉터리 구조 및 동기화 (Structure & Sync)

### **[1순위: 구조] 프로젝트 표준 폴더 구조 및 동기화 (Project Directory Structure & Sync)**
- 모든 프로젝트는 아래의 표준 구조에 따라 폴더를 구성하고 관리하며, 두 경로 간의 상태를 동기화합니다.
- **[중요!] 실제 프로그램 소스 코드 폴더와 에이전트 시스템 공간은 엄격히 분리**합니다. 마스터 계획서, 설계도, 서브에이전트 로그 등 에이전트 기획 산출물은 실제 소스 리포지토리에 저장되어 Git 형상을 오염시키지 않도록 반드시 `.agents/` 시스템 관리 하위 디렉토리에만 보관해야 합니다.

```text
{워크스페이스 루트}/ (Git 제어 공간)
│
├── .agents/ (Antigravity 시스템 관리 공간 - Git 비대상 권장)
│   ├── AGENTS.md (행동 규칙 및 소통 헌법)
│   ├── plugins/ (에이전트 확장 플러그인 폴더)
│   │   ├── harness-plugin/
│   │   └── {도메인별 플러그인들}/
│   │
│   └── projects/ (각 프로젝트 상태 관리 공간)
│       └── {project_name}/
│           ├── STATUS.md (상황판)
│           ├── logs/ (소통 상세 로그)
│           └── _workspace/ (SDLC 폴더 구성 및 AI 산출물 보존)
│               ├── 01_planning/
│               ├── 02_requirements/
│               ├── 03_design/
│               ├── 04_development/
│               ├── 05_testing/
│               └── 06_deployment/
│
└── {project_name}/ (실제 프로그램 소스 코드 공간 - Git 커밋 대상)
```

## 2. 프로젝트별 로깅 및 자산 관리 규칙 (Logging & Assets)

### **[2순위: 투명성] 서브에이전트 소통 상세 로그 기록 (Logging)**
- 메인 에이전트와 서브에이전트 간의 모든 소통(업무 지시, 작업 완료 등)을 개별 `.md` 파일로 작성하여 저장합니다.
- **로그 파일명 작성 규칙:** `YYYYMMDD_HHMMSS_[보낸이]_[받는이]_[작업명].md`
- 이 로그 파일들은 프로젝트 로그 폴더(`.agents/projects/{project_name}/logs/`) 안에 보관 및 관리됩니다.

### **[3순위: 구조] 개발용 보조 스크립트 소스 폴더 보존 규칙 (Script Preservation)**
- 프로젝트 분석, 가공, 데이터 패치 등을 위해 생성한 모든 파이썬 스크립트나 빌드 보조 도구(Utility Scripts)는 임시/스크래치 폴더에 방치하지 않고, 프로젝트의 실제 소스 코드 공간인 `{project_name}/src/` 또는 `{project_name}/scripts/` 하위에 작성하여 영구 보존 및 버전 관리를 보장합니다.

### **[4순위: 효율/구조] 하네스 팀 재사용 및 중복 생성 방지 규칙 (Harness Team Reuse)**
- 각 프로젝트를 시작하기 전 전용 하네스 팀을 정의할 때, 이미 기존에 생성된 유사한 도메인의 하네스 팀 설정이 존재한다면 이를 복제/참조하여 재사용함으로써 에이전트 사양의 불필요한 중복 생성을 막고 설정 관리의 일관성을 높입니다.
